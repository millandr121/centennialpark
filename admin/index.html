Visitor booking mode Controls which booking UI visitors see. Switch anytime — takes effect immediately. Either way, every booking lands in the same Reservations list.

How the two modes work together

📋 Request form — visitors send a request (it lands in Requests). You review, then Accept & assign a site, which creates a confirmed reservation.

🗺 Live booking — visitors pick their own site and dates; a confirmed reservation is created instantly.

Both paths feed the same Reservations backend with the same payment tracking — so you can flip to live booking whenever you're ready with zero data migration.

Admin password
