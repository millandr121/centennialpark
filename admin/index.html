Add manual reservation

Visitor booking mode Controls which booking UI visitors see. Switch anytime — takes effect immediately. Either way, every booking lands in the same Reservations list.

How the two modes work together

📋 Request form — visitors send a request (it lands in Requests). You review, then Accept & assign a site, which creates a confirmed reservation.

🗺 Live booking — visitors pick their own site and dates; a confirmed reservation is created instantly.

Both paths feed the same Reservations backend with the same payment tracking — so you can flip to live booking whenever you're ready with zero data migration.

Admin password

Set via ADMIN_PASSWORD in Cloudflare → Workers & Pages → centennialpark → Settings → Variables.

Data cleanup — delete old records

Permanently deletes records from the D1 database older than the specified number of days. This cannot be undone. Use at end-of-season or to correct mistakes.

Year-end tax / GST export

Downloads a CSV of every paid reservation — category (camping / moorage / parking / launch), amount paid and GST collected — so you can total the GST you owe at year-end and see what it was for.

⚠ Danger zone — wipe the database

Permanently deletes every record — no age limit. For beta testing or a full season reset. You'll be asked to confirm, then to type DELETE. This cannot be undone.
