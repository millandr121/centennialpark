Visitor booking mode Controls which booking UI visitors see. Switch anytime — takes effect immediately. Either way, every booking lands in the same Reservations list.

How the three modes work together

📋 Full form — the complete request wizard (campsite, moorage, parking, launch, add-ons) with a live price estimate. Lands in Requests for you to review and Accept & assign.

✏️ Basic form — a stripped-down request for campsite & moorage only, with no totals or estimates. Same review flow, fewer fields for the visitor.

🗺 Live booking — visitors pick their own site and dates; a confirmed reservation is created instantly.

All three feed the same Reservations backend with the same payment tracking — so you can switch modes anytime with zero data migration.

Data cleanup — delete old records

Permanently deletes records from the D1 database older than the specified number of days. This cannot be undone. Use at end-of-season or to correct mistakes.

⚠ Danger zone — wipe the database

Permanently deletes every record — no age limit. For beta testing or a full season reset. You'll be asked to confirm, then to type DELETE. This cannot be undone.

A full wipe also restarts reference numbers (#1, #2…) from the beginning.

Income & GST report — for taxes

Totals what was actually paid in over a period, split per item (camping, moorage, launch, showers, freezer, donations…), with the GST you collected on each. Pulls from paid bookings and misc items.
